# Content Calendar — Weekly Plan

## Daily Content Types

| Day | Type | Template | Example |
|---|---|---|---|
| Mon | Build Update | Build Update | "Put together a security scanner this weekend. Catches 60 patterns. Claude Code wrote most of it." |
| Tue | Workflow Reveal | Thread (3-5) | "How I set up Claude Code for multi-agent work. Thread:" |
| Wed | Hot Take | Hot Take | "In vibecoding, the code is the easy part. Vision matters. Taste matters." |
| Thu | Tool Discovery | Tool Rec | "Been using [MCP server] for a week. Makes Claude Code actually remember context." |
| Fri | Weekly Metrics | Metric | "This week: 3 skills updated, 1 new pipeline, 2 bugs caught by my own scanner." |
| Sat | Deep Dive | Thread (3-5) | "Why I switched from OpenClaw to Claude Code. Thread:" |
| Sun | Community | Question + replies | Engage with others. Reply to everything. No original posts required. |

## Rules

- Post 1-2 tweets per day (Mon-Sat), not more
- Sunday = reply day only (conversation chains = 150x)
- Never repeat the same template type twice in one week
- If nothing genuine to share, skip the day — silence > filler
- Screenshot/image with every tweet when possible (5-10x boost)

## Monthly Themes

Rotate focus monthly to avoid repetition:
- Week 1-2: Claude Code techniques + skills
- Week 3-4: What you built + results
- Ongoing: tool discoveries, honest fails, community engagement

## Content Sourcing Priority

1. Something that actually happened today/this week (most authentic)
2. A genuine opinion formed from experience (not manufactured takes)
3. A tool you actually use and recommend (not just heard about)
4. Trending topic you can add real perspective to (not bandwagoning)
